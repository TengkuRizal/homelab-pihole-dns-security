# Lessons Learned

## Purpose

This document records operational lessons learned from running Pi-hole as a DNS security and filtering service in a homelab environment.

The goal is to capture practical learning that is useful for DevOps, DevSecOps, and SRE portfolio review.

## Key Lessons

## DNS Is a Critical Dependency

DNS failure can look like internet failure.

Even if the network is working, users may not be able to browse websites if domain resolution fails.

Lesson:

- Treat DNS as a critical internal service
- Monitor DNS availability
- Keep recovery steps documented
- Have a fallback plan

## Visibility Is Valuable

Pi-hole provides visibility into DNS queries generated by clients.

This helps with:

- Troubleshooting client issues
- Understanding outbound domain behaviour
- Identifying noisy devices
- Reviewing blocked domains
- Supporting security investigation

Lesson:

- DNS visibility is useful, but it must be handled carefully because it may contain sensitive information

## Blocklists Need Control

Adding too many blocklists can cause false positives.

False positives can break legitimate websites and applications.

Lesson:

- Add blocklists carefully
- Validate after changes
- Review false positives
- Maintain allowlist process

## Privacy Must Be Protected

DNS logs can reveal browsing behaviour.

Lesson:

- Do not publish raw DNS logs
- Sanitize screenshots
- Hide client names and sensitive domains
- Treat DNS query data as sensitive

## Backup Is Important

Pi-hole configuration can be rebuilt manually, but it takes time and increases recovery risk.

Lesson:

- Export backup after important changes
- Store backup securely
- Test restore process
- Do not commit backup files to public GitHub

## Documentation Improves Operations

Runbooks and design documents make troubleshooting easier.

Lesson:

- Document architecture
- Document recovery steps
- Document risks and tradeoffs
- Keep operational notes updated

## Future Improvements

Planned improvements:

- Add secondary Pi-hole
- Add monitoring and alerting
- Add automated backup
- Add DNS failover testing
- Add sanitized screenshots as evidence
- Add Prometheus and Grafana visibility
